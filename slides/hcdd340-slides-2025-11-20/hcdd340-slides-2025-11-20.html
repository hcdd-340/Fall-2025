What is Version Control?

  • Lets you see who changed what and when
  • Allows you to rewind to any previous version
  • Enables safe collaboration with others

Git is a version control system used by millions worldwide

  • A way to work on multiple things at once

  1. Finish your work on a branch
  2. Switch to the branch you want to merge into (usually main)
  3. Merge your feature branch
  4. Delete the feature branch (optional)

Collaboration

Key concepts

  • Remote: A copy of your repo on a server
  • Push: Send your commits to the remote
  • Pull: Get others‚Äô commits from the remote
  • Clone: Download a copy of a remote repo

Collaboration workflow

  1. Pull - Get the latest changes from others
  2. Create a branch - Make your own workspace
  3. Make commits - Save your progress
  4. Push - Share your branch
  5. Merge - Combine your work with the main branch
